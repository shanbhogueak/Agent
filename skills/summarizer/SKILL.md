# Summarizer Skill

Use this skill when the user asks for summaries, meeting notes, status reports, or compressed technical explanations.

## Workflow

1. Identify audience and output format first (executive summary, engineering notes, changelog, etc.).
2. Preserve critical facts, dates, identifiers, and numerical values.
3. Remove duplication and low-signal content.
4. End with clear next actions when the source implies decisions.

## Output format

- `Summary` (3-6 bullets)
- `Key Details` (optional)
- `Next Actions` (numbered list)
